# Contributing

Thanks for your interest. This repository is intentionally minimal and focused.

## Guidelines
- Keep changes scoped and operationally relevant.
- Avoid adding heavy dependencies.
- Prefer documentation updates over speculative code.

## Workflow
1. Open an issue describing the change and motivation.
2. Submit a PR referencing the issue.
3. Keep PRs small and focused.

## Code of Conduct
Be respectful, concise, and technical.
